### 1-4. `useAuth` 훅 인터페이스 stub 설계

실제 OAuth 연동 전에도 전체 플로우가 동작하도록 인터페이스를 먼저 정의한다. Phase 2에서 내부 구현만 교체한다.

**파일**: `src/hooks/useAuth.ts` (현재 **미존재**)

```ts
// stub 인터페이스 — Phase 2에서 내부 구현 교체
// ⚠️ 기존 코드의 'paid' → 'subscribed'로 통일 필요 (HomePage, CategoryListPage)
export type UserTier = 'guest' | 'free' | 'subscribed';

export interface AuthState {
  isLoggedIn: boolean;
  userTier: UserTier;
  userId: string | null;
  email: string | null;
}

export interface UseAuthReturn extends AuthState {
  login: (provider: 'google' | 'apple', terms: TermsPayload) => Promise<LoginResult>;
  logout: () => Promise<void>;
  withdraw: () => Promise<void>;
  refreshSession: () => Promise<void>;
}
```

**전역 tier / 로그인 상태 통합**: 하드코딩된 목업 값을 `useAuth()`로 교체.

### 2-6. YouTube 임베드 플레이어 구현

## 주요 결정 사항 및 고려 사항

### 결제 플랫폼

- **RevenueCat 사용 권장**: Apple IAP + Google Play Billing을 단일 SDK로 통합하여 구독 상태 관리, 서버사이드 검증, 영수증 처리 복잡도를 크게 줄인다.
- RevenueCat 무료 티어 월 $10,000 이하 수익 구간 무료.
- BE에서 RevenueCat Webhook을 수신하여 구독 상태 동기화.

### 백엔드 우선 개발 항목 (Phase 2 착수 전 필요)

BE 팀이 Phase 1 종료 시점 전후로 준비해야 할 최소 API:

1. `POST /auth/google`, `POST /auth/apple` — 로그인 연동의 전제
2. `GET /user/me` — tier 게이팅의 전제
3. `GET /recommendations/today` — 홈 화면 핵심

### MVP 제외 항목 (출시 후 단계적 추가)

요구사항정의서 기준 MVP 보류 항목:

| 항목 | REQ | 이유 |
|------|-----|------|
| 상황별 추천 | REQ-018 | [보류] MVP UI 범위 제외 |
| 새롭게 발견한 채널 섹션 | REQ-019 | [보류] MVP 제외 |

### iOS 구독 취소 정책

Apple 정책상 앱 내에서 구독 취소 처리가 불가하다. `UnsubscribeBottomSheet`에서 "구독 취소하기" 탭 시 App Store 구독 관리 페이지로 안내하는 방식으로 처리한다.

```ts
// iOS 구독 취소 — 시스템 구독 관리 페이지로 이동
await App.openUrl({ url: 'https://apps.apple.com/account/subscriptions' });
```

### `useSubscription` 훅 구독 취소 플랫폼 분기

```ts
const handleUnsubscribe = async () => {
  if (Capacitor.getPlatform() === 'ios') {
    // Apple 정책: 앱 내 취소 불가 → 시스템 페이지 안내
    await App.openUrl({ url: 'https://apps.apple.com/account/subscriptions' });
  } else {
    // Android: Google Play 구독 관리 또는 인앱 처리
    await App.openUrl({ url: 'https://play.google.com/store/account/subscriptions' });
  }
};
```
